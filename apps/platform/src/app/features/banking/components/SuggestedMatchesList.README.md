# SuggestedMatchesList Component

## Overview

The `SuggestedMatchesList` component displays suggested reconciliation matches generated by the auto-reconciliation service. It allows users to review match confidence scores, matching criteria, and take action to confirm or reject suggested matches.

## Requirements

**Validates: Requirements 9.1-9.10**

- 9.1: Display suggested matches from auto-reconciliation
- 9.2-9.4: Show matching criteria (amount, date, reference)
- 9.5-9.6: Display match confidence scores
- 9.7-9.8: Set reconciliation status to "suggested"
- 9.9: Do not update bank transaction status for suggestions
- 9.10: Present matches for manual confirmation or rejection

## Features

### Display Capabilities

1. **Match List**: Shows all suggested matches with key details
2. **Confidence Scores**: Visual badges indicating match quality (High/Medium/Low)
3. **Matching Criteria**: Badges showing which criteria matched (Amount, Date, Reference)
4. **Transaction Details**: Bank transaction and journal entry information side-by-side
5. **Date Difference**: Shows number of days between transaction and journal entry dates

### User Actions

1. **Confirm Match**: Opens dialog to confirm reconciliation with optional notes
2. **Reject Match**: Opens dialog to reject match with required reason
3. **Auto-refresh**: Reloads matches when filters change

### Filtering

- Filter by bank account ID
- Filter by date range (from/to)
- Automatically reloads when filters change

## UI Components

The component uses the following UI elements:

- **Card**: Container for the match list
- **Table**: Displays matches in a structured format
- **Badge**: Shows confidence levels and matching criteria
- **Dialog**: Confirmation and rejection modals
- **Button**: Action buttons for confirm/reject
- **Textarea**: Input for notes and rejection reasons
- **Alert**: Error message display

## Confidence Levels

The component displays confidence scores with color-coded badges:

- **High (≥90%)**: Green badge - Very likely correct match
- **Medium (70-89%)**: Yellow badge - Probable match, review recommended
- **Low (<70%)**: Orange badge - Uncertain match, careful review needed

## Matching Criteria Badges

The component shows which criteria matched:

- **Amount**: ✓ Amount - Transaction amounts match exactly
- **Date**: ✓ Date (Xd) - Dates match or within tolerance (shows day difference)
- **Reference**: ✓ Reference - Bank reference contains journal entry reference

## Dialogs

### Confirm Dialog

- Shows transaction and journal entry details
- Optional notes field for additional context
- Confirm button to complete reconciliation
- Cancel button to close without action

### Reject Dialog

- Shows transaction and journal entry details
- Required reason field (must provide explanation)
- Reject button (disabled until reason provided)
- Cancel button to close without action

## API Integration

The component uses the `reconciliationService` with these methods:

```typescript
// Get suggested matches
reconciliationService.getSuggestedMatches(
    bankAccountId?: string,
    dateFrom?: string,
    dateTo?: string
): Promise<SuggestedMatch[]>

// Confirm a suggested match
reconciliationService.confirmSuggestedMatch(
    reconciliationId: string,
    notes?: string
): Promise<void>

// Reject a suggested match
reconciliationService.rejectSuggestedMatch(
    reconciliationId: string,
    reason: string
): Promise<void>
```

## State Management

The component manages the following state:

- `matches`: Array of suggested matches
- `isLoading`: Loading state for initial data fetch
- `error`: Error message if loading fails
- `confirmDialogOpen`: Confirm dialog visibility
- `rejectDialogOpen`: Reject dialog visibility
- `selectedMatch`: Currently selected match for action
- `confirmNotes`: Optional notes for confirmation
- `rejectReason`: Required reason for rejection
- `isProcessing`: Processing state during confirm/reject

The component handles errors gracefully:

- Shows error alert if loading fails
