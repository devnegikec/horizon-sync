## Debug Changelog - Test Validation Issue

### Session ID: 20260125_051316_378372

### Problem
Test at line 45 in `ForgotPasswordForm.spec.tsx` failing - `waitFor` not catching validation error for invalid email.

### Root Cause
HTML5's built-in email validation was preventing form submission (firing `invalid` event), which blocked React Hook Form's validation from ever running. This is a common issue when using React Hook Form with HTML5 input types like `type="email"`.

### Solution Applied
Added `noValidate` attribute to the form element to disable HTML5 validation and rely entirely on React Hook Form's validation via Zod schema.
